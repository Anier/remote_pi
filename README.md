# 🤖 Telegram Bot для OpenCode

**OpenCode Telegram Bot** — это прослойка между Telegram и AI-агентом OpenCode, позволяющая отправлять запросы и получать ответы от ИИ прямо в Telegram.

Ответы приходят **токен за токеном** — сообщение в Telegram обновляется в реальном времени по мере генерации текста, как в режиме TUI (терминальный интерфейс) OpenCode.

## Архитектура

```
┌─────────────────┐     Telegram Bot API      ┌──────────────────────┐
│  Telegram User   │ ◄──────────────────────► │  Node.js Bot         │
│  (@your_bot)     │     (long polling)        │  (src/index.js)      │
└─────────────────┘                            └──────────┬───────────┘
                                                           │ HTTP (REST)
                                                           │ Basic Auth
                                                           ▼
                                                  ┌─────────────────┐
                                                  │ opencode serve   │
                                                  │ localhost:4096   │
                                                  ├─────────────────┤
                                                  │ AI Models:       │
                                                  │ GPT, Claude,     │
                                                  │ Gemini, etc.     │
                                                  └─────────────────┘
```

Бот работает в связке из двух процессов:

1. **OpenCode Server** (`opencode serve`) — HTTP-сервер, предоставляющий API к AI-моделям
2. **Telegram Bot** (`node src/index.js`) — связывает Telegram с OpenCode

Оба процесса запускаются локально на одном компьютере. Сервер OpenCode доступен только через localhost (не выставляется наружу), а Telegram Bot использует long polling для получения сообщений.

## Описание компонентов

### `src/index.js` — Точка входа

Главный файл бота. Реализует команды:
- `/start`, `/help` — информация и инструкции
- `/code <запрос>` — отправка запроса ИИ
- `/stop` — немедленная остановка генерации ответа
- `/new [имя]` — создание новой сессии (с опциональным заголовком)
- `/model [provider/model]` — выбор модели для текущего пользователя
- `/models` — получение списка всех доступных моделей с сервера
- `/sessions` — список последних сессий с указанием рабочих папок
- `/switch <id>` — переключение контекста на существующую сессию
- `/session [id] [-f]` — детальная статистика сессии. С флагом `-f` присылает историю в формате `.md`.
- `/projects` — список всех проектов, известных серверу
- `/danger <on|off>` — управление режимом автоматического подтверждения команд

## Команды Telegram-бота

| Команда | Описание |
|---------|----------|
| `/code <запрос>` | Отправить запрос активной модели |
| `/stop` | Остановить текущий ответ ИИ |
| `/new [имя]` | Начать новый диалог (имя опционально) |
| `/model <id>` | Сменить модель (например, `/model google/gemini-2.5-pro`) |
| `/models` | Показать список всех доступных на сервере моделей |
| `/sessions` | Список доступных сессий для переключения |
| `/switch <id>` | Переключиться на указанную сессию по её ID |
| `/session [id] [-f]` | Инфо о сессии. Добавьте `-f` для скачивания истории файлом. |
| `/projects` | Список проектов на сервере |
| `/danger <on/off>` | Включить/выключить авто-подтверждение (RESTART) |

## Запуск

Бот запускается через кросс-платформенный Node-супервизор (`scripts/start.js`),
который поднимает `opencode serve`, ждёт его готовности и затем стартует Telegram-бота.
Супервизор корректно гасит дочерние процессы по `Ctrl+C` и автоматически перезапускает
связку при переключении режима `/danger on|off`.

### Любая ОС (рекомендуется)

```bash
npm start                       # обычный режим
npm run start:skip-permissions  # режим авто-подтверждения (как /danger on)
npm run start:bot               # только бот, без перезапуска opencode serve
```

Или напрямую:

```bash
node scripts/start.js [--skip-permissions] [--no-serve]
```

### Linux / macOS

```bash
./start-bot.sh [--skip-permissions] [--no-serve]
```

### Windows (PowerShell)

```powershell
.\start-bot.ps1 [-SkipPermissions] [-NoServe]
```

PowerShell-скрипт теперь — тонкая обёртка над Node-супервизором, поэтому хардкод путей
больше не нужен: всё работает из любой директории, где находится репозиторий.

**Переменные окружения для тонкой настройки:**
* `OPENCODE_PORT` (по умолчанию `4096`)
* `OPENCODE_HOST` (по умолчанию `127.0.0.1`)
* `OPENCODE_BIN` (по умолчанию `opencode` / `opencode.cmd` на Windows)
* `TELEGRAM_THROTTLE_MS` — пауза между обновлениями Telegram-сообщения (по умолчанию `1000`)
* `SSE_IDLE_TIMEOUT_MS` — idle-таймаут SSE-подписки в мс (по умолчанию 5 мин)
* `TELEGRAM_ALLOW_ALL=true` — намеренно открыть бот всем пользователям (по умолчанию fail-closed)

## Устранение неполадок

### Ошибка: Unauthorized (401)
Это означает, что пароль в `.env` (`OPENCODE_SERVER_PASSWORD`) не совпадает с тем, который сгенерировал сервер.
**Решение:** Перезапустите сервер через `.\start-bot.ps1`, пароль в .env обновится автоматически.

## Дополнительные возможности

### Работа с контекстом
Вы можете переключаться между разными проектами, используя `/sessions` и `/switch`. Бот подхватит историю сообщений и рабочую директорию соответствующей сессии.

### Выгрузка истории
Если вам нужно прочитать историю старой сессии, используйте `/session <ID> -f`. Бот сформирует и пришлет Markdown-файл со всеми репликами, мыслями ИИ и логами инструментов.
