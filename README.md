# 🤖 Telegram Bot для OpenCode

**OpenCode Telegram Bot** — это прослойка между Telegram и AI-агентом OpenCode, позволяющая отправлять запросы и получать ответы от ИИ прямо в Telegram.

Ответы приходят **токен за токеном** — сообщение в Telegram обновляется в реальном времени по мере генерации текста, как в режиме TUI (терминальный интерфейс) OpenCode.

## Архитектура

```
┌─────────────────┐     Telegram Bot API      ┌──────────────────────┐
│  Telegram User   │ ◄──────────────────────► │  Node.js Bot         │
│  (@your_bot)     │     (long polling)        │  (src/index.js)      │
└─────────────────┘                            └──────────┬───────────┘
                                                           │ HTTP (REST)
                                                           │ Basic Auth
                                                           ▼
                                                  ┌─────────────────┐
                                                  │ opencode serve   │
                                                  │ localhost:4096   │
                                                  ├─────────────────┤
                                                  │ AI Models:       │
                                                  │ GPT, Claude,     │
                                                  │ Gemini, etc.     │
                                                  └─────────────────┘
```

Бот работает в связке из двух процессов:

1. **OpenCode Server** (`opencode serve`) — HTTP-сервер, предоставляющий API к AI-моделям
2. **Telegram Bot** (`node src/index.js`) — связывает Telegram с OpenCode

Оба процесса запускаются локально на одном компьютере. Сервер OpenCode доступен только через localhost (не выставляется наружу), а Telegram Bot использует long polling для получения сообщений.

## Установка и настройка

### 1. Настройка `.env`

Создайте файл `.env` в корне проекта:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather

## Описание компонентов

### `src/index.js` — Точка входа

Главный файл бота. Реализует команды:
- `/start`, `/help` — информация и инструкции
- `/code <запрос>` — отправка запроса ИИ
- `/new [имя]` — создание новой сессии (с опциональным заголовком)
- `/model [provider/model]` — выбор модели для текущего пользователя
- `/models` — получение списка всех доступных моделей с сервера
- `/sessions` — список последних сессий с указанием рабочих папок
- `/switch <id>` — переключение контекста на существующую сессию
- `/session` — детальная информация о текущей сессии и рабочей папке

### `src/user-store.js` — Настройки пользователей

Позволяет сохранять выбор модели индивидуально для каждого `chatId`. Данные хранятся в `data/users.json`. Если пользователь не выбрал модель, используются значения из `.env`.

## Команды Telegram-бота

| Команда | Описание |
|---------|----------|
| `/code <запрос>` | Отправить запрос активной модели |
| `/new [имя]` | Начать новый диалог. Если указано имя, сессия будет именованной. |
| `/model <id>` | Сменить модель (например, `/model google/gemini-2.5-pro`) |
| `/models` | Показать список всех доступных на сервере моделей |
| `/sessions` | Список доступных сессий для переключения |
| `/switch <id>` | Переключиться на указанную сессию по её ID |
| `/session` | Инфо о текущей сессии, модели и рабочей папке |
| `/new` | Сбросить контекст (создать новую анонимную сессию) |

Скрипт автоматически завершает старые процессы бота и сервера, проверяет порты и запускает всё заново:

```

## Устранение неполадок

### Ошибка: Unauthorized (401)
Это означает, что пароль в `.env` (`OPENCODE_SERVER_PASSWORD`) не совпадает с тем, который сгенерировал сервер.
**Решение:** Перезапустите сервер через `.\start-bot.ps1`, пароль обновится автоматически.

### DeprecationWarning: punycode
Это внутреннее предупреждение Node.js 21+. Оно не влияет на работу бота. Для подавления используйте `$env:NODE_OPTIONS="--no-deprecation"` перед запуском.

## Дополнительные возможности

### Работа с контекстом
Вы можете переключаться между разными проектами, используя `/sessions` и `/switch`. Бот подхватит историю сообщений и рабочую директорию соответствующей сессии.
